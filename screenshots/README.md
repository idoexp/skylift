# Screenshots

This folder hosts the visuals shown in the README. Replace the placeholders with real screenshots before publishing.

Recommended captures (each 1280×720 minimum, PNG):

| File | Subject |
|---|---|

| `wizard-connection.png` | First step of the wizard: connection form (host, port, user). |

| `modal.png` | The custom modal (e.g. confirm purge manifest) with its smooth animations. |
| `dark-light.png` | Side-by-side of dark vs. light theme to highlight the theme-aware design. |

Tips for shooting:

- Use a clean workspace name (no real client name) and a fake server hostname.
- Hide any sensitive paths in the Output panel before capturing.
- For animated demos, prefer `.gif` (≤ 5 MB) or `.webm` over a static `.png`.
- Light theme captures help non-dev users who often run light themes.

## Animated demo (optional)

A short `.gif` or `.webm` showing the 60-second wizard flow is gold for the marketplace listing and the README hero. Aim for ≤ 15 seconds, 800px wide, optimized to ≤ 3 MB.

Tools:
- [LICEcap](https://www.cockos.com/licecap/) — free, simple GIF recorder for Windows/macOS.
- [Peek](https://github.com/phw/peek) — Linux equivalent.
- [ScreenToGif](https://www.screentogif.com/) — Windows, more controls.
